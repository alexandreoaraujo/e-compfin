Anatomy of an HTML element

tags: used to create elements

ex.: <tag> content </tag>

element: part of a webpage. may contain a data item or a chunk of text or a image, or perhaps nothing.

ex.: <p>This is an element</p>

<p> = opening tag

"This is an element" = content

</p> = closing tag

attributes: contain extra info about the element that I don't want to appear in the actual content

ex.: <p> class= "editor-note"> content </p>

"class" is the attribute name and "editor-note" is the attribute value

note: it is recommended that you quote "" all attribute values, as it makes the code more consistent and understandable

nesting elements: putting elements inside other elements

ex.: <p>This is an <strong>nested </strong> element</p>

Anatomy of an HTML document

<!DOCTYPE html> : its a required preamble. however, they don't do much and are basically just needed to make sure your document behaves correctly.

<html></html>: this element wraps all the content on the entire page. also includes the lang attribute, setting the primary language of the page.

<meta charset="utf-8">: this element sets the character set the document should use to UTF-8. there is no reason to not set this, and it can help avoid some problemns.

<meta name="viewport" content="width=device-width">:the viewport element ensures the page renders at the windth of viewport, preventig mobile browsers from rendering pages wider than the viewport and then shrinking them down.

<title></title>: sets the title of the page,wich is the title that appears in the browser tab.

<body></body>: contains all the content that I want to show to web users when they visit my page.

images: it embeds an image into my page. it does this via the src (source) attribute, which contains the path to our image file.

ex.: <img src="path to image" alt="descriptive text for users who cannot see the image" />

Marking up text

headings: headings elements allow to specify that certain parts of my content are headings -or subreading. HTML contains 6 heading levels, <h1> to <h6>

headings should not be used to make text bigger or bold. Try to create a sequence of heading.

paragraphs: this element contains paragraphs of text.

ex.: <p> this is a single paragraph </p>

lists: consists of at least 2 elements:

  • <ul> are for unordered lists, where the order of the items doesn't matter.
  • <ol> are for ordered lists, where the order of the items does matter.

each iten inside the lists is put aside an <li> (list item) element
